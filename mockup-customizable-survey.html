Customizable Satisfaction Survey — UX Mockup

One configurable multi-question survey per workspace. Primary delivery: WhatsApp Flow — a single in-WhatsApp form, one submit, no chatty per-question messages. Better UX, higher completion, no drop-off mid-sequence. Coexists with the default 5-star CSAT for workspaces without a custom survey. Conversational template sequence kept as a documented fallback only.

✅ Design decision: WhatsApp Flow · engineering to enable Meta capability if needed

Conversational templates vs WhatsApp Flow

Conversational WhatsApp Flow
Customer experience Series of chat messages — chatty, can feel like spam Single in-WhatsApp form — feels intentional, professional
Messages sent 5+ (one per question; NPS needs 2) 1 trigger + the Flow itself
NPS 0–10 Awkward — split into 2 list messages (10-row cap) Native scale on a single screen
Editing the survey Each question change needs Meta re-approval (24h+) One Flow approval; question wording can change without re-approval
Drop-off risk Higher — customer can stop mid-sequence; partial responses All-or-nothing submit — cleaner data, no partials
Engineering effort Template-per-question + multi-message response collection One Flow JSON spec + one webhook with all responses

What's new in this design

1
Active custom survey row — clearly marks the active template with an ACTIVE chip. Description shows question count + types so admins know what their customers are getting without opening the builder.
2
Default fallback row — the existing 5-star CSAT stays available as a read-only fallback. Workspaces that never create a custom template keep their current behaviour. Cannot be edited (it's the platform default).
3
Per-question card (MVP-trimmed) — type chip + required/optional rule chip (rule is fixed: free text is always optional, all others required), language tabs (EN / PT-BR / ES) for admin question text, type-specific options editor, single Delete action. No reorder arrows in v1 (delete + re-add to change order). Option labels and UI strings auto-translate to the customer's app language.
4
Live WhatsApp preview — shows the customer's view as a sequence of template messages with their expected responses. Critical for admins to see how their long survey actually plays out on the customer's side before committing.
5
Survey summary card in the conversation thread — same role as today's 5-star bubble but multi-row. One glance to read all four answers without scrolling the question/response pairs above. Designed to scan quickly for managers / QA.

Key WhatsApp constraint to call out in the ticket: WhatsApp templates have UI limits (max 3 reply buttons, list messages capped at 10 rows). NPS 0–10 (11 options) needs to be split into two list-message templates (0–5 and 6–10), as shown in the preview. Each question type also requires its own Meta-approved template — admins can't change question text or options after Meta approves without re-submitting for review (24h+).
