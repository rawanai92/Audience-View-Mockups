Segment Sharing — UI Polish

Five small fixes to align the Create Segment dialog with the rest of the app's design system, plus a couple of related signals that are missing today.

Fix 1

"Share with" block — clean up & align with the design system

Today's fieldset, padding, and checkbox don't match the rest of the app. The "Everyone in workspace" description copy also says "every admin" when it should say "every user." Proposed cleanup below.

Before — what ships today
SHARE WITH
Only me

You'll be the only person who can see this segment.

Specific users

Pick teammates who should also see this segment.

Queue

Everyone assigned to one of these queues will see the segment.

Everyone in workspace

Every admin in your workspace will see this segment.

✓
Also share matching contacts

People in the audience above will also be able to see all contacts that match this segment.

1Fieldset legend sits inside a border that crashes through the "SHARE WITH" label on the left.
2Excessive vertical padding on each radio card — twice the spacing of comparable lists in the app.
3Checkbox style mismatch — filled-orange square with white check feels heavier than the radio buttons above it.
4Misleading copy on "Everyone in workspace" — says "every admin" when it should be "every user."
After — proposed
Share with
Only me

You'll be the only person who can see this segment.

Specific users

Pick teammates who should also see this segment.

Queue

Everyone assigned to the selected queues will see this segment.

Everyone in workspace

Every user in your workspace will see this segment.

✓
Also share matching contacts

People in the audience above can also see and act on contacts that match this segment.

1Remove the fieldset border. Use a plain caps label ("SHARE WITH") above the options — same pattern as other sections in the app.
2Tighter row padding (12px vertical) — matches the existing list-row density.
3Checkbox uses the same size and visual weight as the radios above it. Light fill on the row matches the muted-row pattern used elsewhere for the toggle.
4Copy fixed to "Every user in your workspace will see this segment."
Fix 2

Specific users picker — fall back to email when name is missing

Some accounts are created via email before they complete signup, so they have no display name yet. Today the picker shows "Unknown user" for those — which looks like a bug and provides no way to identify them. Render the email until a name is collected.

Fix 3

Save confirmation — success modal on segment create

Save currently fires silently. Use the same success modal pattern as the contact-creation flow so the user gets an explicit confirmation of what was created and who it was shared with.

1Same shape as the system-wide "Contact created" modal — header tile + title + sub-line + close X, body with green-check confirm line + detail card, footer with secondary "Close" + primary "View segment".
2Header subtitle adapts to the share mode: "… shared with the Rawan Queue" for Queue, "… shared with Maram, Sofia" for Specific users, "… shared with everyone in your workspace" for Workspace, "… visible only to you" for Only me.
3Detail card surfaces what the user just chose: name, type (Dynamic / Static), share target, and the resulting match count. Easy at-a-glance verification.
