Header Bell — Notification Dropdown

Frontend MVP for the header notification surface. V1 is Reminders only — the dropdown consumes /coreleads/notifications filtered to REMINDER_FIRED events and renders them with the same action parity as the live toast. Other notification types stay invisible until they have product-defined UX of their own.

State 1 — Default (open)

Bell open · unread reminders + recent activity

Click the bell → standard design-system popover anchored under the icon. Unread reminders sit at the top with a left dot indicator and a soft accent background. Toast-parity actions (Mark Done · Snooze) live on each reminder row.

1
Bell badge. Count of currently-active reminders (fired and not yet acted on). Sourced from the backend feed filtered to REMINDER_FIRED. Mirrors the existing badge pattern used elsewhere in the app.
2
Active row treatment. Soft accent background + left dot. Every row in the dropdown is an actionable reminder.
3
Toast parity actions. Each row carries the same Mark Done / Snooze buttons the toast shows, calling the same handlers. Marking Done removes the row from the dropdown (no read-history concept) and decrements the bell badge.
4
Snooze sub-menu. Same duration presets as the toast (15 min / 1 hour / Tomorrow 9 AM / Custom). Opens inline; standard design-system menu. Snoozing also removes the row — it re-appears when the new fire time hits.
5
The name on each row is the conversation contact (the customer / lead the reminder is anchored to) — not the reminder creator. Clicking the row body navigates to that conversation.
6
Reminders only in V1. Frontend filters the feed to REMINDER_FIRED events. Other backend notification types stay invisible until they have product-defined UX of their own. No "View all notifications" footer — no destination page exists yet.

Scope discipline

In / out of scope

In scope (V1 — Reminders only)
  • ✓ Bell click → dropdown
  • ✓ Badge with active-reminder count
  • ✓ Rows w/ Mark Done + Snooze (toast parity)
  • ✓ Mark Done removes the row from the dropdown
  • ✓ Row click → conversation
  • ✓ Empty + loading states
  • ✓ Existing design-system popover
  • ✓ EN / PT-BR / ES copy
Out of scope
  • ✗ Non-reminder notification types (transfers, mentions, completions, etc.)
  • ✗ "View all notifications" page / notification management module
  • ✗ Read-history (Done = gone, not archived in-dropdown)
  • ✗ Filtering / search inside the dropdown
  • ✗ Notification preferences (per-type mute)
  • ✗ Real-time push (WebSocket) — polling is enough
  • ✗ Mobile / small-viewport layout
  • ✗ Net-new component primitives
