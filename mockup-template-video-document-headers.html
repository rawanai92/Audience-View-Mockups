WhatsApp templates — video & document headers

Follow-up to 86exujrb0, which carved out video and document headers as out of scope. Behavioral notes

  • Header type picker — single component shared by Campaigns and Conversations create forms (parity continues from prior epic).
  • Upload only — no hosted URL field. Same as image headers today. File is stored as the template's header sample and reused on every send.
  • Variable substitution — applies to the body only; header media is fixed per template, no per-send variation.
  • Approval gate — Meta reviews the uploaded sample. Templates remain disabled until Approved, same as today.
  • Render parity — Campaigns preview and Conversations composer use the same render component for video/document bubbles. No drift between surfaces.
  • Languages — header media is language-agnostic; body translations (EN / PT-BR / ES) pick the right variant per contact.
  • Replace — opens the picker to switch to a different approved template entirely; not a media-only swap.
  • Validations (per Meta) — video: MP4 up to 16 MB. Documents: PDF, Word, Excel, PowerPoint up to 100 MB. Enforced client-side on drop and on browse-select; backend re-checks at upload. Same error treatment as image today (red dropzone border + helper text).
  • Doc icon badge — file extension shown in the WhatsApp bubble so recipients can tell formats apart at a glance. Reuses the existing neutral document tile; no per-format color coding in scope.
